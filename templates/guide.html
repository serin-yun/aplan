    사용 가이드

    5단계 프로세스
    1. 엑셀 파일 준비

      mapping.xlsx 템플릿을 다운로드하거나 기존 파일을 준비합니다.

    2. 데이터 입력

      objects 시트와 relations 시트에 데이터를 입력합니다.

      • objects 시트: 오브젝트 정보 (object_key, name, system_type, object_type, layer 등)
      • relations 시트: 오브젝트 간 관계 정보 (from_key, to_key 등)
    3. 데이터 로딩

      웹 인터페이스에서 파일을 업로드하여 데이터베이스에 로딩합니다.

    4. 검색 및 탐색

      오브젝트를 검색하고 상세 정보를 확인합니다.

    5. 전체 맵 보기

      시스템 전체 구조를 시각적으로 확인합니다.

    파일 형식 가이드
    objects 시트 필수 컬럼:
    컬럼명 설명 예시
    object_key 업무상 유일키 IF_APLAN_SALES_001
    name 표시 이름 APlan 매출 인터페이스
    system_type 시스템 타입 SAP, APLAN, BW, BI, EAI, IAM, NPD, CDP, LEGACY
    object_type 오브젝트 타입 IF, TABLE, VIEW, JOB, REPORT, UI
    layer 레이어 Legacy, API, Cust, In, Staging, Out, UI
    relations 시트 필수 컬럼:
    컬럼명 설명 예시
    from_key 출발 오브젝트의 object_key SAP_VBRP
    to_key 도착 오브젝트의 object_key IF_APLAN_SALES_001
    상세 문서 다운로드
    자주 묻는 질문 (FAQ)

    네, 파일명은 반드시 mapping.xlsx여야 합니다. 다른 이름으로 저장하면 업로드 시 오류가 발생합니다.

    네, 시트명은 정확히 objects와 relations이어야 합니다. 대소문자를 구분합니다.

    object_key는 유일해야 합니다. 중복된 경우 업로드 전 검증 단계에서 오류가 발생합니다.

    relations 시트의 from_key와 to_key는 반드시 objects 시트에 존재하는 object_key 값이어야 합니다. 존재하지 않는 경우 해당 관계는 스킵됩니다.
