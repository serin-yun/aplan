# 테스트 시나리오 가이드

## 🚀 서버 실행

```bash
# 1. 데이터 로딩 (처음 한 번만 또는 데이터 업데이트 시)
python load_mapping.py

# 2. 서버 실행
uvicorn app.main:app --reload
```

서버가 실행되면 http://localhost:8000 에서 접속 가능합니다.

---

## 📍 테스트 가능한 URL 목록

### 메인 페이지
- **대시보드**: http://localhost:8000/dashboard
- **루트 경로**: http://localhost:8000/ (대시보드로 자동 리다이렉트)

### 검색 및 탐색
- **검색 페이지**: http://localhost:8000/objects
- **전체 시스템 맵**: http://localhost:8000/objects/overview
- **오브젝트 상세**: http://localhost:8000/objects/{id} (예: /objects/1)

### 파일 관리
- **파일 업로드**: http://localhost:8000/upload
- **사용 가이드**: http://localhost:8000/dashboard/guide

### API 엔드포인트
- **API 문서 (Swagger)**: http://localhost:8000/docs
- **API 문서 (ReDoc)**: http://localhost:8000/redoc
- **자동완성 API**: http://localhost:8000/objects/api/autocomplete?q=매출
- **통계 API**: http://localhost:8000/dashboard/api/stats

## 🧪 테스트 시나리오

### 시나리오 1: 대시보드 및 프로세스 가이드 테스트

**URL**: http://localhost:8000/dashboard

**테스트 단계**:
1. 대시보드 접속
   - 통계 카드 확인 (전체 오브젝트, 관계, 시스템 타입, 모듈 수)
   - 프로세스 5단계 카드 확인
   - 각 단계의 상태 확인 (완료/진행중/대기)

2. 프로세스 단계 확인
   - 1단계: 엑셀 파일 준비 (파일 존재 여부에 따라 상태 변경)
   - 2단계: 데이터 입력 (가이드 링크 클릭)
   - 3단계: 데이터 로딩 (파일 업로드 링크 클릭)
   - 4단계: 검색 및 탐색 (검색 링크 클릭)
   - 5단계: 전체 맵 보기 (맵 링크 클릭)

3. 통계 차트 확인 (데이터가 있을 경우)
   - 시스템 타입별 파이 차트
   - 레이어별 바 차트
   - 오브젝트 타입별 도넛 차트
   - 모듈별 가로 바 차트

4. 빠른 작업 카드 클릭
   - 각 카드 클릭하여 해당 페이지로 이동 확인

5. 최근 조회한 오브젝트 (데이터가 있을 경우)
   - 테이블에서 오브젝트 클릭하여 상세 페이지 이동

**예상 결과**:
- 모든 통계 정보가 정확히 표시됨
- 프로세스 단계가 현재 상태에 맞게 표시됨
- 차트가 정상적으로 렌더링됨
- 모든 링크가 정상 작동함

---

### 시나리오 2: 파일 업로드 및 검증 테스트

**URL**: http://localhost:8000/upload

**테스트 단계**:
1. 파일 업로드 페이지 접속
   - 현재 파일 존재 여부 확인

2. 드래그 앤 드롭 테스트
   - 엑셀 파일을 업로드 영역에 드래그
   - 드래그 오버 시 영역 색상 변경 확인
   - 파일 선택 버튼 클릭하여 파일 선택

3. 파일 검증 테스트
   - 올바른 파일 (mapping.xlsx) 업로드
     - 검증 성공 메시지 확인
     - 검증 상세 정보 확인 (시트, 컬럼, 행 수)
   - 잘못된 파일명 업로드
     - 오류 메시지 확인
   - 잘못된 형식 파일 업로드
     - 오류 메시지 확인

4. 파일 업로드 및 데이터 로딩
   - 검증 통과 후 "파일 업로드 및 데이터 로딩" 버튼 클릭
   - 로딩 스피너 확인
   - 성공 메시지 확인
   - 대시보드로 자동 이동 확인

5. 가이드 링크 확인
   - "상세 가이드 보기" 버튼 클릭하여 가이드 페이지 이동

**예상 결과**:
- 드래그 앤 드롭이 정상 작동
- 파일 검증이 정확히 수행됨
- 오류 메시지가 명확하게 표시됨
- 데이터 로딩이 성공적으로 완료됨

---

### 시나리오 3: 스마트 검색 및 필터링 테스트

**URL**: http://localhost:8000/objects

**테스트 단계**:
1. 기본 검색 테스트
   - 검색어 입력 없이 "검색" 버튼 클릭
   - 전체 오브젝트 목록 표시 확인

2. 자동완성 테스트
   - 검색창에 "매출" 입력
   - 자동완성 드롭다운 표시 확인
   - 제안 목록에서 시스템 타입, 오브젝트 타입 배지 확인
   - 키보드 화살표 키로 이동 테스트
   - Enter 키로 선택 테스트
   - 자동완성 항목 클릭하여 검색 실행

3. 검색어 하이라이트 테스트
   - "매출" 검색 실행
   - 결과 테이블에서 "매출"이 하이라이트된 것 확인
   - object_key와 name 필드에서 하이라이트 확인

4. 필터 테스트
   - 시스템 타입: "SAP" 선택
   - 오브젝트 타입: "IF" 선택
   - 검색 실행
   - 필터링된 결과 확인

5. 필터 프리셋 테스트
   - 필터 설정 후 "현재 필터 저장" 클릭
   - 프리셋 이름 입력 (예: "SAP 인터페이스")
   - 저장된 프리셋 버튼 확인
   - 프리셋 버튼 클릭하여 필터 적용 확인
   - 프리셋 삭제 버튼 테스트

6. 활성 필터 칩 테스트
   - 필터 적용 후 활성 필터 칩 표시 확인
   - 필터 칩의 X 버튼 클릭하여 필터 제거 확인

7. 컨텍스트 도움말 테스트
   - 각 필드 옆 "?" 아이콘에 마우스 오버
   - 툴팁 표시 확인

**예상 결과**:
- 자동완성이 실시간으로 작동
- 검색어가 정확히 하이라이트됨
- 필터가 정확히 적용됨
- 프리셋이 저장되고 불러와짐
- 도움말이 표시됨

### 시나리오 4: 전체 시스템 맵 및 네트워크 그래프 테스트

**URL**: http://localhost:8000/objects/overview

**테스트 단계**:
1. 전체 맵 페이지 접속
   - 통계 정보 확인
   - 필터 컨트롤 확인

2. 필터 테스트
   - 시스템 타입 필터 선택
   - 레이어 필터 선택
   - 모듈 필터 입력
   - "필터 적용" 버튼 클릭
   - 필터링된 네트워크 확인

3. 검색 및 하이라이트 테스트
   - 검색창에 오브젝트명 입력
   - "하이라이트" 버튼 클릭
   - 검색된 노드 강조 확인
   - 연결된 노드 반투명 처리 확인
   - 나머지 노드 매우 반투명 처리 확인

4. 네트워크 조작 테스트
   - 노드 드래그하여 이동
   - 마우스 휠로 확대/축소
   - 노드 클릭하여 상세 페이지 이동
   - 노드 더블클릭하여 포커스
   - "전체 보기" 버튼으로 전체 네트워크 보기

5. 레이아웃 옵션 테스트
   - "물리 엔진 토글" 버튼 클릭
   - "계층 레이아웃" 버튼 클릭
   - "레이어별 레이아웃" 버튼 클릭
   - 각 레이아웃의 차이 확인

6. 미니맵 테스트
   - "미니맵 토글" 버튼 클릭
   - 미니맵 표시 확인
   - 미니맵에서 클릭하여 메인 네트워크 이동 확인
   - 메인 네트워크 이동 시 미니맵 업데이트 확인

7. 범례 테스트
   - 시스템 타입별 색상 범례 확인
   - 범례 배지 클릭하여 필터링 확인

**예상 결과**:
- 네트워크가 정상적으로 렌더링됨
- 모든 인터랙션이 부드럽게 작동
- 필터가 정확히 적용됨
- 미니맵이 정상 작동
- 레이아웃 옵션이 정상 작동

### 시나리오 5: 오브젝트 상세 페이지 테스트

**URL**: http://localhost:8000/objects/{id} (예: /objects/1)

**테스트 단계**:
1. 상세 페이지 접속
   - 검색 결과에서 오브젝트 클릭하여 이동
   - 또는 URL 직접 입력

2. 기본 정보 확인
   - 좌측 패널에서 모든 필드 확인
   - 컨텍스트 도움말 아이콘에 마우스 오버
   - 툴팁 표시 확인

3. 영향도 분석 확인
   - Upstream 리스트 확인
   - Downstream 리스트 확인
   - 리스트 항목 클릭하여 다른 오브젝트로 이동

4. 인터랙티브 그래프 테스트
   - "인터랙티브 그래프" 탭 확인
   - 탐색 깊이 변경 (1단계/2단계)
   - 노드 드래그, 확대/축소
   - 노드 클릭하여 다른 오브젝트로 이동
   - 중심 오브젝트(노란색) 확인

5. Mermaid 다이어그램 테스트
   - "Mermaid 다이어그램" 탭 클릭
   - Mermaid 코드 확인
   - "Copy Mermaid Code" 버튼 클릭
   - 클립보드 복사 확인
   - 실제 다이어그램 렌더링 확인

6. 네비게이션 테스트
   - "← 검색으로 돌아가기" 버튼
   - "전체 맵 보기" 버튼
   - 브레드크럼 네비게이션

**예상 결과**:
- 모든 정보가 정확히 표시됨
- 그래프가 정상적으로 렌더링됨
- 네비게이션이 정상 작동
- Mermaid 코드가 정확히 생성됨

### 시나리오 6: 모바일 반응형 테스트

**테스트 방법**: 브라우저 개발자 도구에서 모바일 뷰로 전환

**테스트 단계**:
1. 대시보드 모바일 뷰
   - URL: http://localhost:8000/dashboard
   - 햄버거 메뉴 버튼 확인
   - 메뉴 클릭하여 모바일 네비게이션 확인
   - 통계 카드가 세로로 배치되는지 확인
   - 프로세스 단계 카드가 모바일에 맞게 조정되는지 확인

2. 검색 페이지 모바일 뷰
   - URL: http://localhost:8000/objects
   - 검색 폼이 모바일에 맞게 조정되는지 확인
   - 테이블이 가로 스크롤 가능한지 확인
   - 자동완성 드롭다운이 모바일에서 정상 작동하는지 확인

3. 전체 맵 모바일 뷰
   - URL: http://localhost:8000/objects/overview
   - 네트워크 그래프가 모바일 화면에 맞게 조정되는지 확인
   - 터치 제스처로 확대/축소 가능한지 확인
   - 미니맵이 모바일에서도 표시되는지 확인

4. 파일 업로드 모바일 뷰
   - URL: http://localhost:8000/upload
   - 업로드 영역이 모바일에 맞게 조정되는지 확인
   - 버튼 크기가 터치하기 적절한지 확인

**예상 결과**:
- 모든 페이지가 모바일 화면에 맞게 조정됨
- 터치 인터랙션이 정상 작동
- 텍스트와 버튼이 읽기 쉽고 클릭하기 쉬움

### 시나리오 7: 브레드크럼 네비게이션 테스트

**테스트 단계**:
1. 각 페이지에서 브레드크럼 확인
   - 대시보드: 홈만 표시
   - 검색: 홈 > 검색

   - 파일 업로드: 홈 > 대시보드 > 파일 업로드
   - 사용 가이드: 홈 > 대시보드 > 사용 가이드

2. 브레드크럼 클릭 테스트
   - 각 브레드크럼 항목 클릭하여 해당 페이지로 이동 확인

**예상 결과**:
- 모든 페이지에 브레드크럼이 표시됨
- 현재 위치가 정확히 표시됨
- 클릭 시 해당 페이지로 이동

## 💡 테스트 팁

1. **샘플 데이터 생성**
   ```bash
   python create_sample_mapping.py
   python load_mapping.py
   ```

2. **브라우저 개발자 도구 활용**
   - F12로 콘솔 열기
   - 네트워크 탭에서 API 요청 확인
   - 모바일 뷰로 전환하여 반응형 테스트

3. **에러 확인**
   - 콘솔에서 JavaScript 에러 확인
   - 네트워크 탭에서 실패한 요청 확인
   - 서버 로그 확인

4. **성능 테스트**
   - 대량 데이터로 테스트 (100개 이상 오브젝트)
   - 네트워크 그래프 렌더링 속도 확인
   - 자동완성 응답 속도 확인

## 🐛 알려진 이슈 및 해결 방법

### 이슈 1: 차트가 표시되지 않음
**원인**: Chart.js 라이브러리 로드 실패
**해결**: 브라우저 콘솔 확인, 네트워크 탭에서 CDN 로드 확인

### 이슈 2: 자동완성이 작동하지 않음
**원인**: JavaScript 오류 또는 API 오류
**해결**: 콘솔에서 에러 확인, API 엔드포인트 직접 테스트

### 이슈 3: 미니맵이 표시되지 않음
**원인**: 네트워크가 아직 초기화되지 않음
**해결**: 네트워크 로딩 완료 후 미니맵 토글
